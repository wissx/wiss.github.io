Github + Hexo 搭建个人博客

前言

Hexo 新手写的新手教程，记录自己搭建 Hexo 的过程，所有信息均来自互联网

Hexo 官方中文教程：https://hexo.io/zh-cn/docs/ （不太懂，只看了一点）

对 Hexo 执行的终端命令都是在 blog 目录下执行（cd blog）

网站配置文件：./blog/_config.yml

主题配置文件：./blog/themes/landscape/_config.yml

我是在 Mac 上搭建的

目录

一、安装 Git 和 Node.js

下载 Git 并安装：https://www.git-scm.com/downloads

下载 Node.js 并安装：https://nodejs.org/en/

打开终端执行命令：

git --version

node -v

npm -v

如果都出现版本号，说明安装成功。

二、安装 Hexo

终端执行命令（会在用户目录下创建 blog 文件夹，可自定义）

sudo npm install -g hexo-cli

hexo init blog

cd blog

npm install

hexo generate

hexo server

最后两条可简化成一条 hexo s -g（本地部署）

看到 “INFO Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.” 字样就表示已经生成网页文件并开启了本地服务器，现在浏览器打开 http://localhost:4000 预览博客。（重设端口：hexo -p 5000 server）

三、注册 GitHub

登陆 GitHub 账号，点击右上角用户头像旁的 + 新建仓库 new repository，仓库名称 Repository name 必须为”你的GitHub用户名.github.io”，如 wissx.github.io。（这里我犯错了，导致出现问题）

复制你的仓库地址，例如：https://github.com/wissx/wissx.github.io.git

用代码编辑器（推荐 Sublime）打开网站配置文件 _config.yml，编辑最下面的 deploy 代码

repository 后面是你的仓库地址，其他一致，注意空格。

打开终端在 blog 目录执行命令

hexo generate

hexo deploye

上面两条可简化成

hexo g

hexo d

顺利的话可以成功访问你的博客地址 https://wissx.github.io

若执行 hexo g 出错则执行 npm install hexo --save，若执行 hexo d 出错则执行 npm install hexo-deployer-git --save，再执行 hexo g 和 hexo d。

但我执行 hexo d 后遇到一个接一个的错误，网上查方法解决，一直出现各种不同的错误，搞了两三个小时，都疯了。

如果你碰到错误，看一下代码意思，复制代码在网上查解决方法，真不能急，因为没用。

我最后成功解决的方法是：
把 GitHub 仓库名改成相同的用户名，我原先并非一致。
在 GitHub 帐户设置里取消邮件隐私保护 Keep my email address private。
删除 blog 目录下隐藏文件夹 .deploy_git。

再执行

hexo clean

hexo g

hexo d

清除缓存并部署服务器，然后打开 https://wissx.github.io 成功。

RSS 复兴 - 从认识到使用

不知你是否听过 RSS，相信多数人是没听过的，这是一种古老的技术，这几年快绝迹了，最近一直在喊的“RSS 复兴”纯属意淫，在各大巨头封闭内容圈用户的生态下，内容创作者转向微信公众号的时代里（微信公众号不提供 RSS），还有人知道 RSS 就不错的了，使用人数更是少，下面我来普yi及yin一下。

一、什么是 RSS

RSS 使用 XML 标准来共享内容，把网站的内容聚合一起输出给接收方，简单的说就是内容提供方打包文章内容推送出去，我们订阅一个 RSS 地址，就能获取内容了。
RSS 内容可以是文章、视频、音频、微博动态流等各种形式的内容，一般是指文章。

用人话来说就是在一个地方看不同网站的内容，这个地方是指 RSS 阅读器。

RSS 的流行是由 Google Reader 带起来的，最火的时候应该是08到12年，我是在 Google Reader 13年关闭后才接触 RSS 的，之前并不知道这个东西，我那个时候用的是鲜果阅读器，在14年鲜果也关闭了，微博微信流行起来后，RSS 就落寞了，用户越来越少，我是用过了才知道 RSS 那么的好用，也一直用到现在，订阅的内容不多，才20个左右，不过每天都能收到文章更新，真好。

二、什么是 RSS 服务商和阅读器

RSS 服务商就是收集各种网站提供的 RSS 内容，用户订阅后就可以接收内容，还可以多设备多平台同步。

RSS 阅读器是用来阅读文章的，登陆 RSS 服务商的帐号，订阅内容就会同步到阅读器里。

RSS 服务商会有官方的阅读器，用来登陆自家帐号接收订阅内容并阅读。

很多第三方 RSS 阅读器可以不登陆服务商帐号，用本地订阅 RSS，本地订阅不能同步，我是不会选择的。

有些第三方 RSS 阅读器可以登陆多个服务商帐号，不同的服务商会有不同的服务和功能，用同一个阅读器则可以保证一致的阅读体验。

服务商是用来订阅内容的，阅读器是用来看文章的。

服务商和阅读器的功能会有重叠，之间的关系有一些混乱。

三、用 RSS 服务商 Feedly 来订阅内容

Feedly 是个值得推荐的 RSS 服务商，也是我在用的 RSS 服务商，支持各大平台。
有优秀的官方阅读器，还有许多第三方 RSS 阅读器。

在这个页面可以看到支持 Feedly 的平台和阅读器 Feedly App Center。

在这个页面可以看到用于代替 Google Reader 的阅读器 Alternatives to Google Reader for all platforms

注册 Feedly 登陆后，直接在 Feedly App 搜索栏搜索想要订阅的网站，网页版则是点左下方的添加内容来搜索想要订阅的网站，或者输入网站的 RSS 地址来订阅，提供 RSS 的网站都会把 RSS 地址放在显眼的地方，在网站的最下方、右边或其他页面可以找到，一般和微信微博放在一起。

我推荐两个网站 少数派、利器 给大家订阅，当然也可以订阅我的博客 弋山客，但我很久才更新一次，都不好意思推荐，囧。

想订阅更多内容，可以找自己喜欢的网站、博客，或上网搜索“值得订阅的RSS”，订阅数量不易过多，一天更新几十篇文章的也不要订阅，内容太多看不完会厌恶，只订阅最值得订阅的，每天把内容看完。

对于 RSS 我得说一句，用习惯了微信公众号，改用 RSS，还要自己去找网站的 RSS 地址，相信没多少人愿意学习这么复杂的操作，而且很多网站已经不提供 RSS 输出了，落寞是正常的。
不过有一些小众网站，并没有公众号，坚持更新自己的网站并提供 RSS 输出，这是值得订阅的。

还有个RSS 服务商也得推荐一下，就是 Inoreader，也是个非常优秀的 RSS 服务商，有很多人推荐，不可替代的 RSS，比 Feedly 更好用的 Inoreader，听说功能上比 Feedly 更强大，我没有用过，不过 Inoreader 的第三方阅读器比较少。

Feedly 和 Inoreader 都有官方阅读器，推荐下载使用。

这里有一篇非常专业的文章介绍 Feedly 和 Inoreader 之间的区别 2018 年主流 RSS 服务选哪家？Feedly、Inoreader 和 NewsBlur 全面横评，和这位作者相比，我这种业余选手，只是各种吹水，乱七八糟的凑字数。

四、第三方 RSS 阅读器推荐

第三方阅读器我推荐 Reeder 和 Fiery Feeds，他们都支持 Feedly 和 inoreader，还支持更多服务商和平台。

iPhone 上我在用的是 Fiery Feeds，Mac 上我用的是 Leaf（收费的）。

第三方 RSS 阅读器的操作界面和阅读体验都不相同，得自己用过才知道适合哪个，先用官方的阅读器熟悉使用流程后，再用第三方阅读器来体验区别，慢慢找到自己最喜欢的那款。

安卓的第三方客户端我不熟悉，可以看看我上面两个阅读器介绍页面，代替 Google Reader 的阅读器，支持 Feedly 的阅读器，找一下安卓的，用官方的挺好。

视频、播客、微博我不推荐用 RSS 阅读器来订阅，用起来不方便，还是用视频网站来看视频，微博客户端看微博，播客软件听播客比较方便。

香港旅游交通指南

从深圳去香港在交通行程方面的一些介绍。
我是4月5号从广州坐火车去深圳住，4月6号一早去的香港，当天回的广州。

#提前准备：

  • 通行证
  • 身份证（国内住宿乘车用）
  • 三五百港币和信用卡
  • 移动电源（非常刚需）
  • 淘宝买香港手机卡（13人民币，一天无限流量）口岸自取，到香港后再换到手机里。

#出入境过关流程：
全程走旅客通道和自助通道（E通道）验证通行证和指纹。

#深圳去香港
从深圳罗湖口岸走自助通道出境，到香港罗湖走人工通道入境（第一次），以后走自助通道入境。

#香港回深圳
从香港落马洲走自助通道出境，到深圳福田口岸过两次自助通道入境，不知道为什么深圳这边要验证两次通行证，第一次是通行证+指纹，第二次是通行证。
这是我2018年4月6号出入境的情况。

#香港地铁车票事项：
过关到香港地铁客务中心买八达通（150港币）和游客过境旅游票（100港币，适合罗湖、落马洲坐地铁的游客），淘宝买贵不推荐。

八达通用来小额消费、电车、轮渡之类的很方便，过境旅游票用来长时间坐地铁比较划算，如果少坐地铁那只用八达通即可。

如果回来是坐广九直通车或中港通巴士那就买游客全日通（65港币），可省35港币，都是一天随便无限次坐地铁，区别在于全日通不可用于罗湖站和落马洲站。

坐飞机入境香港的有机场快线旅游票。

#交通路线：
我手机装了四个 APP，谷歌地图、高德地图、12306、Airbnb。

谷歌高德查路线互补用，12306国内买火车票用，Airbnb 订房用。

下载好城市轨道交通路线图到手机相册

这个主屏看着非常普通，一个亮眼的个性的 app 都没有，但使用上没什么不方便，留下最重要的就足够了。

今年的苹果发布会没什么值得讲，观看直播时比往年少了些许兴奋与激动。

iPhone 8 外观没区别，无线充电是个鸡肋，iOS 11 稳定性退步了，AR App 可能是唯一的乐趣。

iPhone X 系统操作逻辑大改，像个工程机，售价近万…

Trello 我的任务管理流程

如果你每天接收各种任务，处理各种事情，这个任务管理/登记流程也许值得一试。

在 Trello 创建一个待办任务列表¹、一个任务封存列表²、几个任务分类列表³。

每天把所有任务添加到待办列表，一个任务一个卡片，然后一个个去做，做完就移动到封存列表。

待办任务当天没做完，下班前就在任务卡片上补充任务要点，这利于之后快速完成任务，并把任务分类，清空待办列表。

第二天继续添加待办任务并处理之前未完成的分类任务。

这套工作流程很像 To-Do List，但有个最大的区别是，这套流程没有铃声提醒功能，不用设置开始时间、完成时间、不用打勾来让任务消失，适合无固定完成时间、每日接任务、在最短时间完成、任务多且繁杂的工作。

团队也可以使用这套流程，接到任务你就把它添加到待办列表，通知团队成员去完成任务，当天没完成的任务下班前就分类整理，并补充详细信息（方便完成任务），添加适合去完成此任务的成员，分类任务就让任务成员来跟进处理。

任务流程这样就走通了。

任务流程演变：

最开始我在备忘录里记任务，发现管理很麻烦。

后来改用 Trello 来分类记录任务，发现在接收任务时就进行任务列表分类是一件很没效率的事。
很多任务当天就能完成，不需要分类，完成后直接移动到封存列表即可。分类需要思考，需要补充信息让其更清晰，不然分类没有意义。

现在把接收到的任务添加到待办列表，当天完成不了，再分类处理，工作效率高很多，操作流程也更简洁了。

  1. 待办列表用作收集当天任务。
  2. 封存列表用作归档以便日后方便查找。
  3. 分类列表用于任务分类，建三个左右比较合适，分类让任务更清晰，同时缓解待办列表的任务数量。

ThinkVision X24 显示器上手

为给 MacBook Air 配一台显示器，看了 2K 显示器，看了 4K 显示器，还看了 21:9 显示器。

4K 是不行的，因为我的 MacBook Air 核显不支持 4K 分辨率。

常看到人说 2K 效果不好，当然价格也是比较贵的，有点鸡肋，便放弃了。

一直想买 21:9 的显示器，合适价格的 21:9 显示器外观都难看，挑了好久，也没看中。（我很喜欢超宽屏）

无意看到一款 1080P 的显示器，被漂亮的外观吸引了，窄边框，超薄机身，性感的底座，主要是价格合适。

618期间正好降价，便剁手了。

显示器箱子很大，有很厚的泡沫，保护得很好，机身很薄，要拿机身下部分或支架才稳妥。显示器配了 HDMI 线和 DP 线，自己买了根 Mini DP 转 DP 线。

双屏显示我的 Mission Control 设置。

美剧，浏览器，刷推，聊天，四个常用操作终于不用在一块 13 寸上频繁切换了，有了一块屏幕分担操作界面，观看体验好多了。

总结就是：眼睛爽。

值得买的苹果产品

苹果产品对我来说

  • 有一些是必要的
  • 有一些是非必要的
  • 有一些是不要的
  • 有一些是想要但没钱的 🙄

我认为自己是果粉，由于勤俭节约（穷）只买过两台 iPhone 和一台 MacBook Air。(一个理性的果粉…)

我认为 Mac 和 iPhone 是最值得买的苹果产品，电脑和手机绝大多数人都有，我为什么选择苹果，最大的原因是苹果产品的体验不流氓。

iPhone 肯定买最新的。Mac 可以多考虑是要台式还是笔记本，我选择了 13 寸的 MacBook Air，足够。

毕竟当时穷，加上自己用不上 Pro 的性能选择了 Air，现在看来 Pro 是更好的选择，因为我的新需求是要增加一个显示器，Pro 有独立显卡，选择显示器的空间大，可以考虑 4K 显示器什么的。

iPad 对我来说就是非必要的，同为 iOS 系统，iPhone 已足够。虽然 iPad 有很优秀的特性，但我不怎么喜欢平板设备。

而 Apple Watch 这种装饰品是我不需要的，现在也就两个功能，健身数据和查看通知，实在没兴趣。

我不喜欢手表、戒指、手链、项链等没什么用的装饰品，戴在身上不自在，甚至包括围巾。

只有 iMac 是想要但没钱买的产品，毕竟不是刚需，因为我已经有 MacBook Air 了。

有一台显示器看中很久了，趁着 618 剁手！
